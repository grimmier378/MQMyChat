MQMyChat Documentation

What is MQMyChat?

MQMyChat is a native C++ MacroQuest plugin that gives you full control over how EverQuest chat is displayed. It is a port of MyChat, originally a standalone Lua script that was later bundled into the MyUI module collection. MyUI is now the primary development home for the Lua version, with backports to the standalone repo. MQMyChat rewrites it in C++ for better performance, deeper integration with MacroQuest, and as a learning project for C++ development. The database format is identical, so existing Lua MyChat users can migrate seamlessly.

Instead of relying on EQ's built-in chat windows (which are limited and hard to filter), MQMyChat lets you create custom chat channels with pattern-based filters, color coding, and multiple independent windows — all rendered through ImGui (a lightweight graphical interface library that MacroQuest uses for drawing windows, buttons, and other UI elements).

Here is what that means in practice:

  • Chat routing: Every line of chat that EQ produces passes through a pattern matcher called Blech (a fast text-matching engine bundled with MacroQuest — think of it as a specialized search tool that watches every line of chat for patterns you define). You write patterns like "#*# tells you,#*#", where #*# is a wildcard that matches any amount of text (see the Blech pattern reference for full syntax). MQMyChat routes matching lines to the channels you choose.
  • Filtered channels: Each channel can have multiple events, and each event can have multiple filters. A filter is a simple string match — if the chat line contains the filter text, it gets routed. You can assign colors per-filter so different types of chat stand out visually.
  • Multiple windows: You can open as many chat windows as you want, each showing a different set of channels as tabs. One window might show all your group/raid chat, another might show tells and auction, and a third might show only spam.
  • Token substitution: Filter patterns can contain tokens like M3 (your character name), GP1 (any group member), N3 (NPC name), and others. At match time, the plugin replaces the token with a live value from the game. This lets you write filters that work across characters without hardcoding names. See the full token reference.
  • Persistence: All your settings — channels, events, filters, window layouts, tab order — are stored in a SQLite database (a small, self-contained database stored as a single file on disk). They survive reloads and can include character-specific or server-specific presets.
  • Lua scripting: An embedded LuaJIT interpreter lets you run custom scripts that respond to chat events, providing extensibility beyond the built-in system.

Source File Structure

The plugin is made up of 11 files. Here is what each one does and roughly how large it is:

File Lines Purpose
MQMyChat.cpp 1065 Plugin entry point. Contains MyChatEngine implementation: chat routing, Blech event setup (including RefreshBlechEvents() for live channel/event toggling), token substitution, command handling, the AppendToConsole() method (which handles timestamp injection), and the hooks into MQ's OnIncomingChat / OnWriteChatColor callbacks.
MQMyChat.h 107 MyChatEngine class declaration. Defines the central engine that ties together the database, renderer, Blech matcher, Lua state, and all channel data. Declares RefreshBlechEvents() and the AppendToConsole() method that all console output routes through.
MyChatChannel.h 73 Core data structures: ChatFilter, ChatEvent, ChatChannel, PresetInfo, MyChatSettings, and important constants like CHANNEL_SPAM and CHANNEL_RESERVED_START.
MyChatDatabase.h 44 MyChatDatabase class declaration. Defines the interface for loading and saving all settings to SQLite.
MyChatDatabase.cpp 832 SQLite persistence implementation. Creates the database schema, handles saving/loading channels, events, and filters, seeds default presets, and manages migration.
MyChatRenderer.h 41 MyChatRenderer class declaration. Defines the interface for the ImGui rendering layer.
MyChatRenderer.cpp 961 ImGui rendering implementation. Draws chat windows, tab bars, the settings/configuration GUI, the channel editor (with live Blech refresh on channel/event toggle), and handles user interaction like scrolling and tab reordering.
MyChatTLO.h 52 Top Level Object (TLO) type declarations. A TLO is how MacroQuest exposes plugin data to macro scripts — think of it as a read-only API that macros can call. Defines MyChatType and MyChatChannelType so macros can query the plugin.
MyChatTLO.cpp 215 TLO implementation. Exposes plugin data to MQ macros and provides the Send method so macros/Lua can inject text into channels.
Theme.h 868 ImGui theme system. Contains 11 built-in color themes that can be applied to the chat windows for visual customization.
CMakeLists.txt 34 CMake build configuration. Links against MQ2Main and required libraries.

Key Dependencies

MQMyChat depends on several libraries, most of which come from MacroQuest itself:

Dependency What it provides
MQ2Main The MacroQuest core. Provides the plugin API, chat hooks, command registration, TLO framework, and character data access.
Blech A fast pattern matching engine bundled with MQ. MQMyChat uses it to match incoming chat lines against event patterns. Patterns use #*# as an uncaptured wildcard (matches any text), and #1#, #2#, etc. as numbered capture wildcards (match any text and pass the matched portion to a callback). These are not single-character matchers — each one matches any amount of text. See the Blech pattern reference for details.
SQLite3 Embedded database engine. All plugin settings, channel definitions, and presets are stored in a SQLite database file. Installed via vcpkg.
LuaJIT / Lua 5.1 Embedded scripting engine. Allows users to write Lua scripts that respond to chat events. The source includes LuaJIT headers directly (luajit/lua.h). Installed via vcpkg.
Zep / ConsoleWidget Zep is a text editor library, and ConsoleWidget is MacroQuest's wrapper around it that provides a scrollable, colored text display area — similar to a terminal or console window. Each MQMyChat channel gets its own ConsoleWidget instance where matched chat lines appear.
ImGui Immediate-mode GUI library (short for "Immediate Mode Graphical User Interface"). Unlike traditional GUIs where you create widgets once and update them, ImGui redraws the entire interface every frame. All of MQMyChat's windows, tabs, buttons, and configuration panels are rendered through ImGui. MacroQuest provides ImGui as part of its core.
fmt Modern C++ formatting library. Used for string formatting throughout the codebase.

How to Build

MQMyChat is built as part of the MacroQuest solution. It is not a standalone project. To build it:

  1. Clone the MacroQuest repository and set up the build environment following the standard MQ build instructions.
  2. Place the MQMyChat folder under plugins/ in your MacroQuest source tree.
  3. Make sure you have sqlite3 and lua51 installed via vcpkg. The CMakeLists.txt uses find_package(unofficial-sqlite3) and find_package(Lua REQUIRED) to locate them.
  4. Add the plugin to the MQ solution (CMake will pick it up if it is in the plugins directory) and build.
Note: If you just want to use MQMyChat, you do not need to build it yourself. Pre-built versions may be available through your MacroQuest distribution. This build section is for developers who want to modify the source.
