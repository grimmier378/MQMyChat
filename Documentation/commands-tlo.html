Commands & TLO API

MQMyChat exposes two interfaces for controlling the plugin from outside its GUI: the /mychat slash command and the ${MyChat} Top-Level Object (TLO). This page covers both in detail so you can drive the plugin from macros, Lua scripts, or the MQ command line.

New to MacroQuest? A slash command is text you type into the EQ chat window starting with /. MacroQuest intercepts it before EQ sees it. A TLO (Top-Level Object) is how macros and Lua scripts read live data from plugins — think of it as a read-only API you access with ${PluginName.Property} syntax.

The /mychat Command

The command is registered during plugin startup inside InitializePlugin() (MQMyChat.cpp, starting at line 26):

PLUGIN_API void InitializePlugin()
{
    g_chatEngine = new MyChatEngine();
    g_chatEngine->Initialize();
    AddCommand("/mychat", MyChatCommand);
    RegisterMyChatTLO();
}

AddCommand is a MacroQuest API that maps a slash command string to a C++ callback function. When you type /mychat in the EQ chat window (or in a macro), MacroQuest intercepts it and calls MyChatCommand() with everything after /mychat as a single string. For example, typing /mychat send Tells hello passes the string "send Tells hello" to the callback.

How Argument Parsing Works

Inside MyChatCommand() (line 101), the plugin uses two MacroQuest helpers to pull arguments out of the string:

  • GetArg(dest, line, n) — copies the nth whitespace-delimited token from line into the dest buffer. Argument numbering starts at 1.
  • GetNextArg(line, n) — returns a pointer to the remainder of line after skipping n tokens. Useful for grabbing a free-form message that may contain spaces.

Each subcommand is matched with ci_equals(), which is a case-insensitive string comparison (so /mychat SHOW works just as well as /mychat show). The first match wins; if nothing matches, the plugin prints a usage message listing every valid subcommand.

Full Command Reference

Subcommand Usage Description
(none) or show /mychat or /mychat show Show the main window. Sets showMainWindow = true.
hide /mychat hide Hide the main window. Sets showMainWindow = false.
send /mychat send <channel> <message> Send a message to the named channel. If the channel does not exist yet, it is created automatically. The message is everything after the channel name.
clear /mychat clear Clear the main console (all text is removed).
clear <channel> /mychat clear Tells Clear a specific channel by name. The name lookup is case-insensitive. If the channel is not found, an error message is printed.
config /mychat config Toggle the configuration GUI on or off. Flips showConfigGUI.
reload /mychat reload Reload settings from the database. Calls UnloadCharacterSettings() followed by LoadCharacterSettings().
lock /mychat lock Lock the window position and size so it cannot be accidentally moved.
unlock /mychat unlock Unlock the window so it can be repositioned and resized again.
preset /mychat preset Default Switch to a named preset. The sequence is: Save current settings, SetActivePreset, Unload settings, Load settings. If the preset name is not found, an error message is printed.

Example: Sending a Message from the Command Line

/mychat send Tells Hey there, how are you doing?

This sends the text "Hey there, how are you doing?" to the channel named "Tells". GetArg grabs "Tells" as argument 2, and GetNextArg(szLine, 2) returns everything after it as the message body.

Example: Switching Presets

/mychat preset Raiding

This saves your current channel settings, switches the active preset in the database to "Raiding", then reloads. All your channel tabs, events, and filters change to match that preset. See the Database & Settings page for more on how presets work.

TLO: MyChat

A Top-Level Object (TLO) lets macros and Lua scripts read plugin state using the ${...} syntax. MQMyChat registers the MyChat TLO so that other tools can inspect channels, check the plugin version, and send messages programmatically.

If you have used MQ2 macros before, you have already seen TLOs like ${Me.Name} or ${Target.Level}. The MyChat TLO works the same way — it gives you a dot-notation path to read plugin data. From Lua, the equivalent is mq.TLO.MyChat.PropertyName().

How the TLO is Registered

In MyChatTLO.cpp, RegisterMyChatTLO() creates two custom MQ2Type subclasses (think of these as data type definitions that tell MacroQuest what properties an object has) and hooks the TLO name:

void RegisterMyChatTLO()
{
    pMyChatChannelType = new MyChatChannelType();
    pMyChatType = new MyChatType();
    AddTopLevelObject("MyChat", MyChatType::dataMyChat);
}

AddTopLevelObject tells MacroQuest: "whenever someone writes ${MyChat...}, call MyChatType::dataMyChat to resolve it." The TLO is unregistered in ShutdownPlugin() via UnregisterMyChatTLO(), which calls RemoveTopLevelObject and deletes both type objects.

Two custom MQ2Type subclasses power the TLO. An MQ2Type is a C++ class that tells MacroQuest how to resolve dot-notation access on an object — for example, what to return when someone writes .Version or .Channel[...]:

  • MyChatType — the top-level type. Handles ${MyChat.Version}, ${MyChat.Channel[...]}, etc.
  • MyChatChannelType — returned when you access a channel. Provides per-channel properties like Name, Enabled, and Echo.

MyChat Type Members

Member Return Type Description
${MyChat} string Returns "MQMyChat" (via the ToString override). Useful for checking if the plugin is loaded.
${MyChat.Version} float Returns 0.1 — the current TLO version number.
${MyChat.Channel[name]} MyChatChannel Look up a channel by its display name (case-insensitive). Returns a MyChatChannel object if found.
${MyChat.Channel[id]} MyChatChannel Look up a channel by its numeric ID. The code tries to parse the index as a number first; if that succeeds, it does an ID lookup instead of a name lookup.
${MyChat.ChannelCount} int Returns the total number of channels currently loaded.

MyChat Type Methods

Method Usage Description
Send ${MyChat.Send[ChannelName,message text]} Sends a message to the named channel. Creates the channel if it does not exist. Returns true on success, false if the input could not be parsed.

How Send Parses Its Input

The Send method receives the entire content between the square brackets as a single string. It splits on the first comma to separate the channel name from the message:

std::string input(Index);
size_t sep = input.find(',');
if (sep != std::string::npos)
{
    std::string channel = input.substr(0, sep);
    std::string message = input.substr(sep + 1);
    // trim leading/trailing spaces from channel name
    // trim leading spaces from message
    g_chatEngine->SendToChannel(channel, message);
}

Key points:

  • The comma is the delimiter. If there is no comma, the method returns false.
  • Whitespace is trimmed from both ends of the channel name and from the start of the message.
  • The message itself can contain commas — only the first comma is treated as the separator.

MyChatChannel Type Members

When you access a channel via ${MyChat.Channel[...]}, you get a MyChatChannel object. Here are the properties you can read:

Member Return Type Description
Name string The channel's display name (e.g., "Tells", "Combat").
ID int The channel's numeric ID.
Enabled bool Whether the channel is active and processing events.
TabOrder int The channel's position in the tab bar (lower numbers appear first).
PopOut bool Whether the channel is displayed in its own separate window instead of as a tab in the main window.
Echo string The command prefix used when typing messages into the channel's input box (e.g., /tell, /gu).
MainEnable bool Whether messages matching this channel are also echoed to the Main tab.

MyChatChannel ToString

If you evaluate a channel object directly (without accessing a member), its ToString returns the channel name. For example, ${MyChat.Channel[0]} might return "Consider".

Lua Script Integration

Lua scripts running inside MacroQuest can use the TLO just like macros can. The MyUI framework's Utils.PrintOutput() function (in MyUI/lib/common.lua) demonstrates the recommended pattern: check if the plugin is loaded, and fall back gracefully if it is not.

The PrintOutput Pattern

PrintOutput() works like this:

  1. Check if the MQMyChat plugin is loaded by evaluating mq.TLO.MyChat().
  2. If loaded, call mq.TLO.MyChat.Send("channel,message") to route the message through MQMyChat.
  3. If the plugin is not loaded, fall back to a Lua-based MyChat handler or print().

This pattern ensures your script works whether or not the user has MQMyChat installed.

Example: Using the TLO from Lua

local mq = require('mq')

-- Check if the plugin is loaded
if mq.TLO.MyChat() ~= nil then
    -- Send a message to a channel
    mq.TLO.MyChat.Send("Tells,Hello from Lua!")

    -- Read channel info
    local ch = mq.TLO.MyChat.Channel("Tells")
    if ch() then
        print("Channel name:", ch.Name())
        print("Channel enabled:", ch.Enabled())
        print("Tab order:", ch.TabOrder())
    end

    -- Check total channel count
    print("Total channels:", mq.TLO.MyChat.ChannelCount())

    -- Look up a channel by numeric ID
    local ch0 = mq.TLO.MyChat.Channel(0)
    if ch0() then
        print("Channel 0 is:", ch0.Name())
    end
else
    -- Plugin not loaded, fall back
    print("MQMyChat not loaded, using default output")
end

Example: Conditional Output Helper

Here is a small helper function you can drop into your own Lua scripts:

local mq = require('mq')

local function sendToChannel(channelName, message)
    if mq.TLO.MyChat() ~= nil then
        mq.TLO.MyChat.Send(channelName .. "," .. message)
        return true
    end
    print(message)
    return false
end

-- Usage
sendToChannel("Combat", "You crit for 9999 damage!")
sendToChannel("Tells", "Reminder: rebuff in 5 minutes")
Note: The Send method splits on the first comma, so your channel name must not contain commas. The message itself can contain commas freely.
How chat reaches MQMyChat: The plugin hooks two MacroQuest callbacks: OnIncomingChat (for chat arriving from the EQ server) and OnWriteChatColor (for text written to the MQ console by plugins). Both feed incoming lines into the Blech pattern engine, which routes them to the appropriate channel. The Send method bypasses Blech routing entirely and writes to the channel's console through AppendToConsole(). This means sent messages also receive timestamps when the Timestamps option is enabled. If the target channel has mainEnable = true, the message also appears in the Main tab.

How SendToChannel Works Internally

Both the Send TLO method and the /mychat send command ultimately call MyChatEngine::SendToChannel(). This method:

  1. Searches through all loaded channels for a case-insensitive name match.
  2. If found, calls AppendToConsole() on the channel's console widget (and on the main console if mainEnable is true). The message goes through the same timestamp-injection logic as all other output.
  3. If the channel name is not found, the method creates a new channel with that name on-the-fly, then appends the message to it. This is how external plugins like MQMyDPS can send data to MQMyChat without any prior configuration — the channel appears automatically the first time a message is sent to it.
void MyChatEngine::SendToChannel(
    const std::string& channelName, const std::string& message)
{
    for (auto& [id, ch] : settings.channels)
    {
        if (ci_equals(ch.name, channelName))
        {
            if (ch.console)
                AppendToConsole(ch.console, message,
                    MQColor(240, 240, 240));
            if (ch.mainEnable && mainConsole)
                AppendToConsole(mainConsole, message,
                    MQColor(240, 240, 240));
            return;
        }
    }

    int newId = GetNextChannelId();
    CreateChannel(channelName, newId);
    // ... append to the newly created channel ...
}
Use case: MQMyDPS integration. The MQMyDPS plugin (a C++ damage parser) can send formatted DPS reports to MQMyChat by evaluating ${MyChat.Send[DPS,formatted report text]}. The first time this is called, MQMyChat automatically creates a "DPS" channel tab. Subsequent messages append to that existing channel. The user can then configure the channel (enable/disable, pop out, change font size) just like any other channel.

Macro (MQ2) Syntax

If you are writing a traditional MQ2 macro instead of Lua, the same TLO works with the standard ${...} syntax:

/echo ${MyChat.Version}
/echo ${MyChat.ChannelCount}
/echo ${MyChat.Channel[Tells].Name}
/echo ${MyChat.Channel[Tells].Enabled}
/echo ${MyChat.Send[Combat,You scored a critical hit!]}
