ImGui Rendering

All of MQMyChat's visual output — the chat windows, tabs, configuration panels, and channel editors — is drawn using ImGui (Immediate Mode GUI). If you have not worked with ImGui before, the key idea is that it is "immediate mode": instead of creating persistent widget objects (like a button that stays around until you destroy it), you describe what should be on screen every single frame. Each frame, your code says "draw a button here, draw text there," and ImGui handles the rest — hit testing, hover effects, click detection, all of it. If your code does not draw the button on the next frame, it simply disappears. This makes the code very straightforward: what you see on screen is exactly what the render function last described.

The rendering code lives in MyChatRenderer.h (class declaration) and MyChatRenderer.cpp (implementation).

MyChatRenderer Class

The renderer exposes 4 public methods, all called from OnUpdateImGui in MQMyChat.cpp every frame:

Method What It Draws
RenderMainWindow() The primary chat window with menu bar, tab bar, and console output
RenderPopOutWindows() Separate ImGui windows for channels that have been "popped out"
RenderConfigGUI() The settings/configuration window where you manage channels
RenderEditChannelGUI() The channel editor window for editing events and filters

Each method is self-contained — it creates its own ImGui window and handles all interaction within it. The call order does not matter.

Main Window

RenderMainWindow() (starting at line 67 in MyChatRenderer.cpp) draws the primary chat interface.

Window Setup

The window is created with:

ImGui::Begin("My Chat - Main##MQMyChat_{charName}", ...);

Key details:

  • First-use size: 600 x 400 pixels, set via ImGui::SetNextWindowSize(ImVec2(600, 400), ImGuiCond_FirstUseEver). The ImGuiCond_FirstUseEver flag means this size is only applied the very first time the window appears. After that, ImGui remembers the size the user dragged it to (stored in the character's overlay INI file, typically config\MacroQuest_Overlay\server_charname.ini) and uses that instead.
  • Window flags: Always includes ImGuiWindowFlags_MenuBar, which tells ImGui to reserve space for a menu bar at the top of the window. When the window is locked, it also adds ImGuiWindowFlags_NoMove and ImGuiWindowFlags_NoResize so the user cannot accidentally drag or resize it.
  • Begin/End pairing: Every ImGui window starts with ImGui::Begin() and must end with ImGui::End(). Everything you draw between those two calls appears inside that window. If Begin() returns false (meaning the window is collapsed or hidden), the renderer skips drawing the contents but still calls End() — ImGui requires this pairing even when the window is not visible.
  • Theme: The active theme is applied before the window begins rendering and reset after it finishes, using ImGuiTheme::ApplyTheme() and ImGuiTheme::ResetTheme().

Tab Bar

Below the menu bar is a tab bar containing one tab per enabled channel, plus a hardcoded "Main" tab that always appears first.

ImGui::BeginTabBar("##ChatTabs", ImGuiTabBarFlags_Reorderable
                                   | ImGuiTabBarFlags_FittingPolicyShrink
                                   | ImGuiTabBarFlags_TabListPopupButton);
About the ## prefix: In ImGui, the ## in an ID string separates the visible label from the internal ID. "##ChatTabs" means there is no visible label (empty string before ##), but ImGui internally identifies this tab bar as "ChatTabs". This prevents ID conflicts when multiple elements share the same visible text.

Key behaviors:

  • Reorderable: You can drag tabs to rearrange them. MQMyChat tracks the new order and saves it (see below).
  • Shrink policy: When there are too many tabs to fit, they shrink to accommodate more. A popup button (small arrow) appears at the edge of the tab bar to list all tabs in a dropdown.
  • Sort order: Channels are sorted by their tabOrder field from sortedChannels. The Main tab is always first.
  • PopOut channels skipped: If a channel has popOut=true, it does not appear in the tab bar — it gets its own window instead.

Tab Reorder Detection

When you drag a tab to a new position, MQMyChat detects the change using DetectTabReorder(). It works by recording the X pixel position of each tab after rendering, sorting them left-to-right, and comparing that order to the previous frame. If the order changed, it marks the tab order as dirty.

FlushTabOrder() then handles the save, but with a deliberate 3-second delay (defined as TAB_ORDER_SAVE_INTERVAL in MyChatChannel.h). This debouncing prevents a flood of database writes when you rapidly drag tabs around. It:

  1. Updates the tabOrder field on each channel to reflect the new order
  2. Saves everything to the database once the 3-second quiet period elapses

ConsoleWidget and Zep

Each channel's chat output is rendered using a ConsoleWidget (mq::imgui::ConsoleWidget), which is built on top of the Zep text editor component.

What Is Zep?

Zep is an embeddable text editor that provides scrollable, syntax-highlighted text display. MQMyChat uses it as a read-only console — you cannot edit the text, but you get smooth scrolling, text selection, and colored output.

Adding Text

When a chat line is routed to a channel (see Chat Routing), it is added to that channel's console via:

console->AppendText(text, color, newline);

The color parameter controls the text color for that line. The newline parameter determines whether a line break is added after the text.

Font Size

Zep manages its own font rendering. Font size is controlled through:

  • SetPixelHeight(size) — Sets the font size in pixels
  • GetPixelHeight() — Reads the current font size back

Zep also supports Ctrl+/- zoom (Ctrl+= to increase, Ctrl+- to decrease, Ctrl+0 to reset) internally, without MQMyChat needing to handle it. These are the number row keys only — numpad keys have no effect.

DrawConsole and DrawMainConsole

DrawConsole() (line 632) and DrawMainConsole() (line 735) handle the layout of each channel's display area.

Layout

The layout is straightforward:

┌──────────────────────────────┐ │ Console Output (Zep) │ │ Takes all available height │ │ minus footer │ │ │ │ │ │ │ ├──────────────────────────────┤ │ > Input field │ └──────────────────────────────┘

The console fills all available vertical space, leaving room for a single-line input field at the bottom.

Input Field

The input field at the bottom of each console supports:

  • Command execution: If the typed text starts with /, it is sent directly to MacroQuest's DoCommand() function. Otherwise, the channel's echo command is prepended. For example, typing "hello" in a channel with echo /gu sends /gu hello.
  • Command history: Use the Up and Down arrow keys to cycle through previously typed commands. Each channel maintains its own history (the Main tab has a separate history too). History is capped at 100 entries (MAX_COMMAND_HISTORY). When you re-enter a command that already exists in the history, the old duplicate is removed and the command is added fresh at the end, so your history stays clean.
  • Local echo: When enabled, your typed text appears in the channel as > text in gray, so you can see what you sent.
  • Auto-focus: After pressing Enter to send a command, the cursor automatically returns to the input field so you can keep typing.

Focus Key

The focus key is a configurable keyboard shortcut that immediately moves your cursor to the input field. The default is RightShift.

CheckFocusKey() maps a key name string (like "RightShift") to an ImGui key enum. Supported keys include:

Category Keys
Modifiers LeftShift, RightShift, LeftCtrl, RightCtrl, LeftAlt, RightAlt
Special GraveAccent (backtick), Enter, Tab, Space, Backspace, Delete, Insert
Navigation UpArrow, DownArrow, LeftArrow, RightArrow, Home, End, PageUp, PageDown
Function F1 through F12

When the configured key is pressed, the renderer sets an internal flag (m_setFocus = true) which triggers ImGui::SetKeyboardFocusHere() on the next frame, placing the cursor in the input field.

Pop-Out Windows

RenderPopOutWindows() (line 264) creates a separate ImGui window for each channel that has popOut = true.

ImGui::Begin("My Chat - {channelName}##MQMyChat_{charName}{channelId}", ...);
  • First-use size: 400 x 300 pixels
  • Each pop-out has its own Lock button (toggles move/resize lock) and Settings button (opens the Edit Channel GUI for that specific channel)
  • Uses the same DrawConsole() rendering as tabs in the main window
  • Closing the window (clicking the X button) sets popOut = false on that channel, returning it to the tab bar

Pop-out windows are useful when you want to keep certain channels always visible regardless of which tab is active in the main window. For example, you might pop out your Tells channel so you never miss an incoming tell.

Config GUI

RenderConfigGUI() (line 303) draws the channel management window.

ImGui::Begin("My Chat - Config##MQMyChat_{charName}", ...);
  • First-use size: 500 x 400 pixels

Channel Table

The main content is a table listing all channels. Each row uses ImGui::PushID(id) / ImGui::PopID() to scope its widgets. This is an ImGui pattern that prevents ID collisions — without it, every "Edit" button would have the same internal ID and ImGui would not know which one you clicked. By pushing the channel ID onto the ID stack, each row's widgets get unique IDs automatically.

The table columns are:

Column Description
EnabledCheckbox to turn the channel on or off
NameThe channel's display name
EchoThe echo command (e.g., /gu)
EventsNumber of events defined for this channel
ActionsEdit button (opens the channel editor) and Remove button
Note: The Spam channel (channel ID 9000) does not appear in this table. It is toggled via the Options menu in the main window's menu bar instead.

Adding a Channel

Below the table is an "Add Channel" form with two input fields:

  • Name — The display name for the new channel
  • Echo — The echo command to use

Clicking Create adds the channel to the in-memory list. A Save All button persists all changes to the database.

Edit Channel GUI

RenderEditChannelGUI() (line 414) opens a dedicated editor for a single channel.

ImGui::Begin("My Chat - Edit: {name}##MQMyChat_{charName}{id}", ...);
  • First-use size: 400 x 300 pixels

Channel Settings

At the top of the editor, you can modify:

  • Name — The channel's display name
  • Echo — The echo command
  • Font Size — Per-channel font size (independent of the main font size)
  • Main Enable — Whether this channel's messages also show in the Main tab
  • Enable Links — Whether clickable item links are active
  • PopOut — Whether to display this channel in a separate window

Events List

Below the channel settings is a list of events, each displayed as a collapsible header (ImGui::CollapsingHeader). The header shows the event index and a preview of the pattern text (truncated to 40 characters). Click it to expand and see the full event details.

Each event shows:

  • Enabled checkbox — Toggle the event on/off without deleting it
  • Remove Event button — Delete the event entirely
  • Pattern input — The Blech pattern this event matches against
  • Event Color picker — Default color for lines matching this event. This is actually the color of filter index 0, which is auto-created if the event has no filters. It serves as the fallback color when no other filter matches.

Filters

Within each event, you can define filters. The first filter (index 0) is special — it serves as the default color for the event and is controlled by the "Event Color" picker above. User-defined filters start at index 1. Each filter has:

  • Pattern input — A string to match within the chat line
  • Color picker — The color to apply when this filter matches
  • Enabled toggle (On) — Turn the filter on/off
  • Hidden toggle — Match the line but suppress its display (see Database & Settings for how this is stored)
  • Remove button (X) — Delete the filter
  • Reorder arrows — Up arrow and down arrow buttons let you move a filter higher or lower in the list. The up arrow appears on all filters except the first user filter, and the down arrow appears on all except the last. Reordering swaps the filter positions and updates their filterIndex values. Filters are evaluated in order, so the position matters — the first matching filter determines the color used for the line.

Add Filter and Token Reference

The Add Filter button includes a tooltip (hover over it) that shows all available tokens you can use in filter patterns. These tokens are placeholders that get replaced with actual character or NPC names at runtime:

Token Meaning
M3Your character name
M1Main assist name
PT1Your pet's name
PT3Any group member's pet name
GP1Party member's name (any group member)
TK1Main tank name
RLRaid leader name
H1Group healer (DRU, CLR, or SHM)
G1–G5Party member in group slot 1 through 5
N3NPC name
P3PC (player character) name

In addition to tokens, two prefix modifiers are available:

Modifier Meaning
NO2Invert match. Place before a token or word to exclude lines that match. For example, NO2GP1 would exclude lines from group members.
^Anchor to start of line. For example, ^You only matches lines that begin with "You".

Lua pattern syntax (%d, %a, etc.) is also supported in filter strings for advanced matching. See Chat Routing & Tokens for a full explanation of how tokens and filters interact.

Add Event

At the bottom of the events list is an input field and button to add a new event. You type a Blech pattern and click Add.

Saving Changes

Clicking Save in the channel editor triggers several actions:

  1. Unregisters all existing Blech events for this channel
  2. Re-registers the events with updated patterns
  3. Sorts the channel list
  4. Saves everything to the database

Font Size Management

MQMyChat has two levels of font size control:

Level Setting Scope
Global settings.mainFontSize Applies to the Main tab console
Per-channel channel.fontSize Applies to a specific channel's console

The valid range is 8 to 48 pixels.

Sync and Apply

Two helper methods manage font sizes:

  • SyncFontSizes() — Reads the current pixel heights back from Zep into the settings structures. This is important because Zep's Ctrl+/- zoom can change font sizes outside of MQMyChat's knowledge. Sync is called before saving so that zoom changes are persisted.
  • ApplyFontSizes() — Pushes the stored font sizes from settings into Zep. Called after loading settings from the database to apply your saved sizes to the console widgets.

How Themes Are Applied

Themes are applied and reset around each window render:

auto oldStyle = ImGuiTheme::ApplyTheme(themeIndex);
ImGui::Begin("My Chat - Main##MQMyChat_...", ...);
// ... render window contents ...
ImGui::End();
ImGuiTheme::ResetTheme(oldStyle);

The ApplyTheme() call saves the current style, then applies the theme's color overrides. It returns the saved style so that ResetTheme() can restore it afterward. This save-and-restore pattern means the theme only affects MQMyChat's windows — other plugins' ImGui windows are not changed.

Theme Picker

The theme picker is accessed through the Options menu in the menu bar. It is drawn by ImGuiTheme::DrawThemePicker(), which shows a dropdown with all available themes. Selecting a theme immediately applies it. The selected theme index is stored in the global_settings table as themeIdx.

Rendering Flow Summary

Here is the complete rendering flow that happens every frame:

OnUpdateImGui() [MQMyChat.cpp] │ ├─ RenderMainWindow() │ ├─ ApplyTheme() │ ├─ Menu Bar (Lock, Settings, Options, Channels, Links, PopOut, Presets) │ ├─ Tab Bar (##ChatTabs, Reorderable) │ │ ├─ "Main" tab → DrawMainConsole() │ │ └─ Channel tabs → DrawConsole() for each (skip popOut channels) │ ├─ DetectTabReorder() → FlushTabOrder() after 3s debounce │ └─ ResetTheme() │ ├─ RenderPopOutWindows() │ ├─ ApplyTheme() (once for all pop-outs) │ ├─ For each popOut channel: │ │ ├─ Lock + Settings (Edit) buttons │ │ └─ DrawConsole() │ └─ ResetTheme() │ ├─ RenderConfigGUI() [if open] │ ├─ ApplyTheme() │ ├─ Channel table (Enabled, Name, Echo, Events, Actions) │ ├─ Add Channel form │ ├─ Save All button │ └─ ResetTheme() │ └─ RenderEditChannelGUI() [if open] ├─ ApplyTheme() ├─ Channel settings (Name, Echo, Font Size, etc.) ├─ Events list with collapsible headers │ └─ Filters per event (pattern, color, toggles, reorder arrows) ├─ Add Event input ├─ Save button → unregister/register Blech events ├─ Close button └─ ResetTheme()

For how the data shown in these windows is stored and loaded, see Database & Settings. For how chat lines arrive and get routed to channels, see Chat Routing & Tokens. For how to add new features to the rendering layer, see How to Extend.
